# PRD — 나에게 맞는 정부지원사업 찾기 (가칭: "딱맞는지원")

> 한 장짜리 MVP 제품 요구사항 문서 · v1.0 · 2026-06-27
> (상세 SaaS 비전은 `PRD-saas-full.md` 참조)

## 한 줄 목적
내 조건(지역·년차·분야·회사 특징)을 입력하면, 그에 맞는 정부지원사업을 **자동으로 필터링해 제시**하는 웹앱. 데이터는 **브라우저 localStorage**에 저장하는 백엔드 없는 클라이언트 앱.

## 사용자
- **IT/SW 분야 초기 스타트업의 대표·기획자** (1차 타깃)
- 지원사업 정보를 일일이 찾아다닐 시간·인력이 부족한 사람
- 로그인·회원가입 없이 내 브라우저에서 바로 쓰고 싶은 사람

## 핵심 기능 (4개)
1. **내 조건 입력** — 지역, 년차(업력), 분야(다중 선택), 회사 특징(자유 서술) 입력·저장.
2. **자동 필터링·매칭** — 입력 조건과 공고 데이터를 대조해 **자격 필터 + 적합도 순 정렬**로 맞는 사업만 제시.
3. **앱 열 때마다 갱신** — 앱 실행 시 데이터를 다시 읽어 신규/마감임박 공고를 반영, 마지막 갱신 시각 표시.
4. **관심 사업 저장** — 마음에 드는 공고를 즐겨찾기, 마감일·요건을 한눈에 관리.

## 화면 구성 (대략)
| 화면 | 내용 |
|------|------|
| ① 조건 입력 | 지역·년차·분야 선택 + 회사 특징 자유 입력 폼, [내 조건 저장] |
| ② 결과 목록 | 적합도 순 공고 카드(공고명·기관·마감일·지원금·적합 배지), 마지막 갱신 시각 |
| ③ 상세 보기 | 공고 상세 + 자격요건 충족/미충족 표시 + 원문 링크 + [관심 저장] |
| ④ 관심 목록 | 저장한 공고 모음, 마감 임박 정렬 |

> 단일 페이지(SPA) 형태로 탭/섹션 전환 가능. 순수 HTML/CSS/JS.

## 저장 방식
- **브라우저 localStorage** 사용 (서버·DB·로그인 없음).
  - `user_conditions` — 내 입력 조건
  - `favorites` — 관심 사업 목록
  - `last_updated` — 마지막 갱신 시각
- **공고 데이터**: 초기엔 **샘플 JSON 파일**(정적)을 번들로 포함해 로드. (실제 Open API 연동은 범위 밖)
- 데이터는 내 브라우저에만 저장됨 — 기기/브라우저 바뀌면 초기화됨(이 점 명시).

## 이번엔 안 만들 것 (범위 밖)
- ❌ 실제 정부 사이트 **크롤링·Open API 연동** (샘플 JSON으로 대체)
- ❌ **AI 제안서 작성**·사업계획서 자동 생성
- ❌ **회원가입·로그인·멀티 사용자**·서버 저장·과금
- ❌ 백그라운드 **타이머·푸시 알림** (앱 열 때 갱신만)
- ❌ HWP 등 **양식 파일 처리**

## 성공 기준 (MVP)
- 조건 입력 → 3초 내 맞는 사업 목록 제시.
- 조건이 좁아 결과가 없으면 **조건 완화 제안** 노출.
- 새로고침/재방문 시 입력 조건·관심 목록이 유지됨(localStorage 동작 확인).
